# 上下文与连续性

## 上下文分层

按当前任务组装最小上下文：

- 稳定层：小说定位、书级合约、世界硬规则、势力与舞台限制、角色硬事实；
- 窗口层：当前卷、节奏段、相邻章节职责；
- 章节层：本章计划、义务合同、读者体验合同；
- 承接层：上一章实际尾段、最近稳定摘要、未完钩子；
- 状态层：当前角色状态、资源、关系、信息边界和伏笔；
- 表达层：风格合约、有限参考样本和用户本次要求。

先保留硬约束，再压缩软信息。不得为了节省上下文删除本章任务、上一章承接、角色硬事实或禁止越界。

## 读取策略

- 写当前章时，不默认读取整本正文。
- 优先读取上一章尾段、最近摘要和与本章冲突直接相关的旧章节。
- 使用拆分角色档案时，先读取 `characters/character-roster.md`，再只读取实际参与角色和高风险角色的独立档案；不要为了一个场景加载全部角色档案。
- 只读取当前场景会触发的世界规则、涉及势力和舞台条目；记录其稳定 ID、当前状态与本章会触发的限制。
- 只读取已到推进窗口或与本章职责相关的伏笔。
- 用户提供参考作品时，提取结构、节奏和表达特征，不长期复制大段原文。
- 迁移后的工作区先用 `continuity_store.py assemble-context` 按章节和参与角色取得有限上下文；SQLite 只给出候选，最终内容必须回读 YAML。索引缺失或过期时直接读取 YAML，不加载全书台账。

## 连续性资产

维护以下事实类别：

- 不可逆剧情事实；
- 角色身份、阵营、能力与位置；
- 角色确认知道和仍不知道的信息，以及参与章节时的可见身份与当下呈现；
- 资源获得、消耗、损坏和所有权；
- 关系状态及其变化原因；
- 未兑现承诺、目标窗口和当前进度；
- 上一章留下的直接钩子责任。
- 已经明确发生的势力控制、舞台准入、公共秩序或规则例外变化。

只有正文明确发生或审查确认的内容才能进入稳定事实。计划中的内容不能提前写成已发生事实。

连续性启用后的权威边界如下：

- 若存在 `continuity/data/`，其中 YAML 是事实、伏笔、资源和动态关系的唯一权威；各 Markdown ledger 是只读视图，SQLite 是可删除索引；

- `fact-ledger.md` 记录不可逆事实；
- `payoff-ledger.md` 记录跨章承诺和伏笔窗口；
- `resource-ledger.md` 记录跨章关键资源；
- 角色档案记录角色自己的状态、资源、知情与直接关系；
- 角色总索引只记录跨角色关系矩阵与共用限制；
- `world-bible.md` 记录世界规则、势力、舞台和它们当前可确认的公共状态；
- `chapter-deltas/` 记录某章如何改变上述资产，但不替代它们的当前汇总；
- `baseline.md` 仅建立首次启用时的可信起点，不补写未生成的逐章历史。

角色档案中，角色自身的硬事实、当前资源、已知与未知信息、可变呈现和一对一关系状态为权威内容；章节计划只摘取参与角色的当下呈现，不复制完整档案。总索引只保存跨角色约束、启用窗口和关系矩阵。`world-bible.md` 只保存会约束后续选择的世界公共状态，不写角色私人情绪或逐章流水账。正文稳定后，更新发生变化的角色档案、必要的关系矩阵和有章节证据的世界状态；不要把计划内容写进这些资产。

## 续写

续写前确认：

1. 最后一个稳定章节；
2. 该章的实际尾段和净变化；
3. 下一章计划是否仍与已发生事实一致；
4. 参与角色当前状态；
5. 必须承接的钩子；
6. 当前卷尚未兑现的阶段承诺。

如果现有计划与正文冲突，优先保护正文事实，再修订未写计划。

## 基线、指纹与 stale

对已有小说首次启用连续性时，选择最近一章稳定正文作为基线章。只读取该章的正文与审查、相关角色档案、当前卷窗口和既有设定，写入当前可确认状态；不要把更早章节逐一重建为 delta。

在 baseline、chapter delta 和 context package 中记录稳定正文的相对路径与 SHA-256 指纹。若正文被用户修改、替换或不存在：

1. 保护正文，不回滚也不自动覆盖；
2. 将依赖它的 delta、关联账本条目、context package 与 recovery 摘要标记 `stale`；
3. 重新从最终正文核对后再提交新的连续性资产；
4. 对受影响的未写计划按常规依赖规则标记 stale。

## 失败处理

- 上下文缺失：明确缺失项，使用保守默认，不臆造精细历史。
- 参考资料读取失败：降级为无参考继续规划，不让主链中断。
- 状态索引与文件冲突：以受保护产物和稳定正文为准。
- 角色事实冲突：先修正角色事实来源或上下文选择，再判断是否需要改正文。
- 伏笔逾期：先作为推进风险，不自动触发全书重规划。
- 质量债：正文可继续时记录章节、证据、风险、建议回收窗口与状态；除非章节职责已结构性失配，不升级为重规划。
