# 职责型并行章节生产（暂停）

> 当前 Skill 不会加载或执行本实验。保留本页仅作历史记录；在任何小说生产任务中不得创建子代理、滚动预热或采用本页候选合同。改用 `chapter-production.md` 的单代理串行链。

## 角色与权限

协调主代理是唯一的生产协调者和工作区写入者。它读取权威资产，冻结快照，创建子代理，检查候选，决定是否修复，并在正文稳定后回灌连续性。

| 角色 | 工作对象 | 必须返回 | 禁止事项 |
| --- | --- | --- | --- |
| 章节策划代理 | 后续第 1、2 章 | 各章 `plan` 与 `context` 候选、相邻章职责风险 | 写入工作区、把预测事件写成事实、改变当前章合同 |
| 正文写作代理 | 当前已冻结章节 | 一份完整、连续的正文候选及自检 | 按场景拆稿、修改计划/账本/状态、宣布定稿 |
| 审查代理 | 当前完整正文候选 | 合同核对、连续性检查、风险分级与处理建议 | 直接修订权威正文、宣布 `accepted`、回灌事实 |

一次最多创建 3 个子代理。通常并行运行“当前章正文写作 + 下一章策划 + 下两章策划”；正文返回后再使用审查代理。若可用并发较少，优先保证当前章正文，其次最近的下一章策划。所有子代理提示必须明确要求：只返回消息候选，不写入任何小说工作区文件。

## 冻结快照与候选合同

协调主代理为每批候选建立只读快照，至少记录：

- `last_committed_chapter` 与当前目标章节；
- 最近稳定正文、`novel-state.yaml`、基线和相关事实/伏笔/资源账本的相对路径与 SHA-256；
- 当前章计划、上下文清单、相关角色与世界条目的版本或哈希；
- 快照时间和当前卷窗口。

每个子代理返回的候选都必须包含以下字段；它们存在于消息中，不写入小说工作区：

```text
role: planner | drafter | reviewer
target_chapter: <正整数>
status: proposed
source_snapshot: <last_committed_chapter、路径与哈希>
input_contract: <计划/上下文及其哈希；审查还含正文哈希>
result: <角色对应的计划、上下文、完整正文或审查结果>
discovered_or_proposed_facts: <事实或“无”>
risks_and_missing_constraints: <列表或“无”>
recommended_handoff: <下一步>
```

策划候选必须把预期事件明确标记为“计划”，不得混入稳定事实。正文候选必须说明是否覆盖篇幅、must-hit、禁止越界、净变化和章末钩子。审查候选只能使用 `recommend_accept`、`recommend_warning`、`patchable_obligation_gap`、`rewrite_needed` 或 `plan_misalignment` 等建议性结论；`accepted` 只能由协调主代理在最终回灌时写入。

## 滚动调度

1. 从恢复记录定位当前章 `N`，确认 `N-1` 是最近稳定章，并建立快照 `S(N-1)`；没有稳定章、状态不一致或存在未解决的阻塞项时，改走标准串行流程。
2. 协调主代理先冻结当前章 `N` 的计划和上下文。并行启动一个正文写作代理处理 `N`，以及至多两个章节策划代理分别为 `N+1`、`N+2` 产生候选。
3. 接收正文候选后，协调主代理检查其输入哈希、章节合同、完整性与明显截断；然后交给审查代理。审查、必要的人味化与修复保持围绕同一份完整正文串行进行。
4. 只有协调主代理可以把通过审查的正文写入章节目录，并按连续性账本顺序更新差分、账本、角色资产、恢复记录和状态。
5. 回灌后重新读取稳定资产并验证 `N+1`、`N+2` 候选的 `source_snapshot`。任何哈希变化、已发生事实冲突、章节号错位、计划职责变化或缺失硬约束，都会使候选失效；丢弃它并从新快照重新策划。
6. 将仍有效的 `N+1` 候选提升为当前章输入后，重复本流程。候选不会因等待而自动成为 `ready` 或稳定资产。

## 接收、失败与回退

协调主代理接收候选时依次检查：角色与章节号是否匹配、状态是否为 `proposed`、快照哈希是否仍匹配、输入合同是否完整、输出是否覆盖该角色责任，以及候选是否试图声明稳定事实或请求直接写入。

下列任一情况都应丢弃候选：代理超时或失败、正文被截断、合同字段缺失、来源文件变化、审查发现硬冲突、或者候选越权修改状态。协调主代理随后从最近稳定资产重新执行本章所需的串行步骤；不得等待失败代理重试而阻塞回灌，也不得把失败候选部分写入账本。

## 运行记录

只在协调主代理的任务报告中说明本批使用的角色、候选是否采用、失效原因和回退情况。不要把瞬时子代理候选保存为新的权威文档；稳定工作区仍只包含章节、连续性资产、状态和既有质量债。
