# 章节生产

## 章节计划合同

在 `plan.md` 中保持同一份合同，供写作、验收和修复共同消费。

### 基本任务

- 本章即时目标；
- 主要阻力；
- 参与角色及各自作用；
- 必须推进的剧情和信息；
- 必须保留的事实与优势；
- 必须触碰的伏笔或承诺；
- 必须发生的角色目标变化；
- 参与角色的当下呈现：本章可见身份、衣着/伤病/气色、可用物件、行为或声音锚点，以及不得误写的状态；
- 本章调用的世界规则、势力和舞台 ID，以及它们造成的准入、资源、信息或行动限制；
- 可以延后的事项；
- 禁止越过的边界。

### 读者体验

- 读者进入本章时最关心的问题；
- 本章可见回报及强度：`setup`、`partial` 或 `major`；
- 主角即时欲望；
- 关键转折；
- 情绪位移；
- 信息交付；
- 章末相对章初的净变化；
- 继承的旧钩子责任；
- 新的章末牵引。

### 章节节奏段与篇幅预算

每份 `plan.md` 都必须写入一份可执行的篇幅合同；没有用户特别指定时，继承 `novel-brief.md` 的书级默认值。字数统计使用**正文汉字/字符数，不含标题、Markdown 标记和审查记录**；它是创作预算，不要求机械凑到精确数字。

- 目标字数：例如 `3000`；
- 可接受区间：例如 `2500–3500`；
- 预算来源：书级默认 / 用户固定 / 本章经批准的例外；
- 本章为何需要该长度：追逐、审讯、情绪转折、信息揭示或多线交叉等；
- 最低有效内容：即使接近下限也必须完成的事件、回报、转折和章末钩子；
- 超出上限时优先删减的内容：回顾、重复解释、同义情绪、无后果的环境描写；
- 未达下限时只能补入的新内容类型：新的有效行动、阻力、选择、对话冲突、可验证信息或关系变化。

在场景预算前，先设计 3—5 个**章节节奏段**。节奏段不是机械的固定百分比，也不等于地点切换；它描述读者在这一段承受的压力如何变化。每一段都写明：进入状态、主角动作与阻力、信息或关系变化、读者回报/情绪、离场状态和字数预算。至少包含：

- 开篇抓手：直接承接上一章问题，让主角立即行动；
- 升级或转折：出现更高代价、新证据、误判或选择；
- 回报与章末牵引：兑现本章承诺的一部分，同时留下下一章必须回应的问题。

场景预算只规划本章实际需要的 2—5 个场景；一个节奏段可含一个或多个场景，但不得把同一件事换句话说成多个段。各段预算之和应接近目标字数；若差距明显，先修计划责任，不用旁白填字数。

```markdown
## 章节节奏段与篇幅预算

- 目标字数：3000
- 可接受区间：2500–3500
- 预算来源：书级默认；本章无覆盖
- 本章长度理由：一次审讯、一次现场查验与一次章末反转，需要完整呈现对峙和证据链。
- 低于下限仍必须完成：A、B、C
- 超长时先删：重复回顾、第二轮无新信息的争执

| 节奏段 | 进入状态 → 动作与阻力 → 转折 | 读者回报/情绪 | 预算 | 离场状态 |
| --- | --- | --- | ---: | --- |
| 开篇抓手 | 旧钩子压迫主角行动 | 紧张、明确问题 | 600–800 | 主角必须作出选择 |
| 升级转折 | 阻力加码并交付新信息 | 局面或关系反转 | 1000–1300 | 原计划失效或升级 |
| 回报与牵引 | 兑现部分承诺，留下新压力 | 可见回报与追读欲 | 700–1000 | 明确净变化和下一章问题 |
```

## 写作上下文优先级

按优先级读取：

1. 本章计划与义务合同；
2. 上一章实际尾段；
3. 角色硬事实与参与角色的当下呈现；
4. 禁止越界与受保护揭示；
5. 当前卷窗口和节奏段职责；
6. 当前场景涉及的世界规则、势力状态和舞台限制；
7. 相关未兑现承诺和资源状态；
8. 风格合约和参考写法。

摘要不能替代上一章实际尾段；软性角色简介不能替代角色硬事实。

## 上下文清单

连续性已启用时，在 `context-packages/chapter-XXX.md` 中为本章建立简明清单。它只记录：

- 本章目标、稳定正文来源与当前卷窗口；
- 已选权威文件和每个文件的用途；
- 必需约束（事实、角色身份与当下呈现、禁止越界、直接钩子、资源风险，以及实际调用的世界条目 ID 和限制）；
- 因无关、重复或超出窗口而裁掉的内容；
- 缺失的硬约束与保守处理方式。

清单不能复制大段正文、完整角色档案或整本设定。缺失硬约束时，不编造历史；缩小本章行动或把风险列入审查。

## 正文生成

1. 从行动、压力或决定点开场。
2. 尽早明确本章即时目标。
3. 通过阻力、复杂化或新信息迫使主角选择。
4. 交付本章承诺的可见回报。
5. 让状态、信息、力量或关系产生净变化。
6. 用威胁、揭示、代价、关系翻转或即将发生的对抗形成章末牵引。
7. 以篇幅合同控制节奏：接近下限但义务未完成时补有效剧情；接近上限仍无净变化时删减或重组，不以总结、回顾或重复心理活动填充。
8. 逐段兑现节奏计划：每一段结束时都应改变主角可做的事、读者知道的事、关系温度或风险等级；不能把全部转折压到结尾。

默认一次生成整章，保持声音、因果和情绪连续。场景卡用于规划、诊断和局部修复，不要默认按场景多轮拼接正文。

## 中文小说反模板化二稿

完整正文生成后，默认进行一次受约束的二稿。先读取 [chinese-novel-humanization.md](chinese-novel-humanization.md)，再执行：

1. 锁定章节合同、已发生事实、信息边界、角色关系、物件归属和章末钩子。
2. 找出成簇出现的模板化表达，而不是逐词清洗。
3. 以场景中的动作、误解、具体物件、人物偏见和未说尽的话替换解释性旁白；允许段落长度与情绪节奏自然变化。
4. 重新检查本章必须发生事件、禁止越界和净变化。

二稿可以重组段落和句子，不必保留初稿段落数。不得用错别字、病句、随机无关细节、隐藏字符、伪造作者经历或篡改人物事实制造“人味”。当用户要求对照实验时，保留初稿并将二稿写入独立文件；未经确认不得覆盖用户编辑的正文。

若用户提供外部检测结果，先读取 [chinese-novel-humanization.md](chinese-novel-humanization.md) 的“外部检测器的有效性边界”。工具未通过与任务相关的对照验证前，不得用其分数触发重写、停损或作者参与流程；作者参与仅由作者对个人创作边界的需求触发。

## 接收判断

先判断：

- 是否存在可读正文；
- 是否完成 must-hit 义务；
- 是否越过 forbidden crossing；
- 主角是否主动；
- promised reward、key turn 和 net change 是否可感知；
- 是否承接上一章；
- 是否出现硬事实冲突；
- 是否处于篇幅合同的可接受区间；若不在区间，是否有明确且可接受的剧情原因，以及是否出现凑字数或仓促收束；
- 是否完成章节节奏段：开篇是否迅速进入压力，中段是否真实升级，回报是否在章末前可见，钩子是否承接本章净变化；
- 是否有明显占位符、AI 自述、重复、截断或工程词泄漏。
- 是否存在成簇的模板化叙事：过密否定式并列、格言化总结、每段都刻意收束、平均化短段、解释代替场景、角色口吻趋同。

不要将外部检测器的百分比当作来源证明或唯一验收标准。将其作为可记录的观察值，仍以章节合同、可读性、人物辨识度、连续性和用户的创作选择为主。

把缺失义务区分为：

- `patchable_obligation_gap`：局部补写即可；
- `rewrite_needed`：整章组织需要调整；
- `plan_misalignment`：本章职责与邻章安排失配。

篇幅偏差本身不是自动重写理由。正文过短但本章义务缺失，按缺失义务处理；正文过长且存在重复或无效段落，优先局部删改；正文已完成合同且偏差来自用户确认的风格选择，可记录为 `continue_with_warning` 而非强行改写。

## 稳定后更新

正文稳定后再更新：章节摘要、已发生事实、角色状态、资源变化、关系变化、信息边界、伏笔推进，以及已经在正文中明确发生的世界状态变化。不要在正文仍处于修复时提前写入这些资产。

稳定的含义是：审查为 `accepted`，或为 `continue_with_warning` 且正文仍可作为下一章的事实来源。此时按以下顺序执行：

1. 记录 `continuity/chapter-deltas/chapter-XXX.md`，并标明最终正文来源与指纹。
2. 更新事实、伏笔和资源账本；个人资源、知情边界与直接关系同时回写相关角色档案和必要的角色矩阵。若正文明确改变势力控制、舞台准入、公共秩序或规则的已知例外，在 `world-bible.md` 的“当前世界状态”中追加带章节证据的状态项；不得把角色猜测、计划事件或未证实传闻写成世界事实。
3. 将可继续的局部问题写入 `production/quality-debt.md`；不得把它们伪装成已修复。
4. 更新 `production/recovery.md` 与 `novel-state.yaml` 的最后已提交章节和唯一下一步。

`local_patch_plan`、`rewrite_needed` 与 `stop_for_replan` 不是稳定结果。它们不得产生稳定事实、资源变更或伏笔兑现记录。
