# 贡献指南

感谢你希望改进 Ani Book Skill。我们尤其欢迎能让创作过程更清晰、更可恢复、更尊重作者决定的贡献。

## 开始前

1. 先搜索已有 Issue，避免重复工作。
2. 对较大的工作流、数据格式或行为调整，先创建 Issue 说明问题、目标和影响范围。
3. Fork 本仓库，创建聚焦单一目的的分支。

## 贡献范围

- 修正工作流中的矛盾、遗漏或难以执行的说明。
- 为脚本补充标准库测试、错误处理或跨平台兼容性。
- 完善使用文档与经授权的脱敏示例。
- 改进连续性、章节审校和创作节奏方面的可操作规范。

请不要提交未获授权的小说正文、他人作品、个人信息、访问令牌或付费资料。涉及现有小说文本的 Issue 和 PR 应使用最小必要片段，并说明授权情况。

## 本地检查

本项目目前只使用 Python 标准库。提交前请至少运行：

```powershell
python -m compileall scripts
python scripts/export_novel_txt.py --help
python scripts/check_continuity_workspace.py --help
```

如果修改了工作区格式或连续性逻辑，请用一个临时或已授权的小说工作区运行连续性检查，并在 PR 中描述结果。

## Pull Request 要求

- 一个 PR 解决一个清晰的问题，标题说明用户可见的结果。
- 保持 Markdown 易读；不要为无关内容重新排版。
- 更新受影响的 README、参考规范和更新记录。
- 新增行为应附带复现步骤、测试或可审阅的示例。
- 接受维护者可能要求的修订、拆分或文档补充。

参与本项目即表示同意遵守 [行为准则](CODE_OF_CONDUCT.md)，并同意你的贡献按 [Apache License 2.0](LICENSE) 发布。
