# 跨书知识图谱：当前方向、已开发内容与待开发项

> 状态：2026-07-20。本文是跨书资产与知识图谱能力的开发看板；面向使用者的已发布变化仍以 [更新记录](../releases/release-notes.md) 为准。

## 当前开发方向

项目保持 **Codex 原生**：Codex 是唯一负责理解、创意判断、关系判断、事件提议、规划、写作与审校的引擎；Skill 和 references 是过程合同；Python 仅承担确定性校验、状态变更、来源追溯、冲突检测、JSONL/SQLite 派生索引和导出。

跨书知识图谱的目标不是从正文自动“猜测事实”，而是把已经验收、可回读、可追溯的资产组织为有限图谱候选，服务于长篇小说的稳定续写和共享 IP 治理。YAML、已验收 Markdown 与其证据仍是权威源；SQLite 只是可丢弃、可重建的索引。项目不引入模型 Provider SDK、Web API、数据库权威、队列或自研 Agent Runtime。

当前主线是：

```text
已验收正文 / 已提交连续性 / 作者确认资产
  -> Codex 提出资产、关系、章节选择或正史事件
  -> 确定性校验与来源指纹
  -> JSONL / SQLite 派生图与有限查询
  -> Codex 回读权威源后写作或审校
  -> 定稿后形成下一轮可审阅候选
```

## 已开发内容

### 基础生产与连续性

- schema v3 小说工作区、显式迁移与备份、唯一下一步、里程碑审批、章节范围内串行生产。
- YAML 连续性事实、角色、资源、伏笔、关系、检查点和可重建 SQLite 索引；用户编辑和已验收正文受到保护。
- 章节计划、有限上下文、正文、润色、审校、修复、连续性提交、稳定正文导出与 Token 用量账本。

### 第一期：私有跨书资产图谱

- Git 忽略的私有 `libraries/`，分别保存 `reusable` 可复用资产和 `universe` 共享 IP 资产。
- 稳定 ID、版本、内容指纹、来源章节/工件/证据、治理状态和可见范围。
- `fork` 独立导入与 `sync` 受保护链接；共享更新只标记 `update_available`，本书冲突只标记 `conflict`，从不静默覆盖正文或本书快照。
- 资产库与单书工作区均可生成 `nodes.jsonl`、`edges.jsonl` 和可重建 SQLite 图索引；图查询只提供有限邻域候选。

### 第二期：资产进入章节生产链

- Codex 可为每章提出 `context-packages/chapter-XXX.assets.yaml`，记录资产用途、约束、导入模式和锁定版本/哈希。
- `novelctl context` 消费经过校验的选择清单，在总预算内加入跨书候选，并输出必须回读的权威路径。
- 已选 `sync/conflict` 会阻断上下文、正文和审校；`update_available` 固定使用本书快照并提示，不自动升级。
- 仅正文已验收且连续性已提交后，才能创建带实际工件 SHA-256 与证据的资产发布候选。

### 第三期：单宇宙编年史、影响审查与正史治理

- 一个私有资产库默认对应一个共享 IP 宇宙；旧库兼容地以 `library_id` 作为 `universe_id`。
- 已发布的 `universe/event` 以稳定 `sequence`、参与资产、影响摘要和可选 `precedes` / `follows` 派生正史编年史。
- `canon-check` 校验已发布同宇宙端点、来源证据与已提交连续性、顺序矛盾、循环关系和失效资产；`timeline` 稳定展示事件顺序。
- `impact` 只检查命令显式传入的 schema v3 工作区，报告受影响的同步链接、章节选择和时间线邻域，且始终返回只读结果。
- 支持作者批准、按资产 Codex 委托与可撤销的宇宙级 Codex 委托；宇宙级委托不会改变 `reusable` 资产治理。

## 待开发项

以下是下一阶段的候选工作，按对“可用知识图谱”的直接价值排序；均须保持 Codex 提议、脚本校验、作者可审阅和不覆盖受保护内容的边界。

1. **正史审查工件与决策记录**
   - 将事件候选、`canon-check` 结果、`impact` 报告和作者决定收敛为可保存、可回读的审查包。
   - 记录“批准、退回、保持本书分叉、撤销委托”等显式决策，形成正史变更审计链。

2. **按时间线的章节上下文风险提示**
   - 将已审但尚未发布的正史影响，以只读风险形式关联到相关章节选择和上下文包。
   - 仍不自动升级快照、不自动改变选择清单；只有既有的实际 `sync/conflict` 继续阻断生产。

3. **面向作者的图谱查询与追溯视图**
   - 提供按角色、势力、道具、事件、书籍和来源章节的有限邻域、路径与证据查询。
   - 每条查询结果必须附带版本、状态、证据路径和回读提示，图谱结果不能直接替代权威事实。

4. **资产生命周期与安全迁移**
   - 明确资产退役、替代、归档、分叉回收与影响复核流程。
   - 为较早的资产库和工作区提供显式迁移、备份、校验及 SQLite 丢失后的重建演练。

5. **跨库导出与协作边界（后续选择项）**
   - 设计由作者显式触发的脱敏导出包、导入校验和来源保留；不把私有 `libraries/` 直接变成在线共享服务。

## 暂不纳入当前路线

- Neo4j、Qdrant、外部向量服务、在线协作平台、Provider 模型调用、Web API、数据库权威和自研 Agent Runtime。
- 从未验收正文自动抽取事实、自动判定正史、自动解决冲突、批量覆盖跨书快照或正文。

这些能力不是当前知识图谱成立的前提。当前优先级是先让每个节点、关系、事件和影响结论都能回到明确的 YAML/Markdown 证据与作者决策。
